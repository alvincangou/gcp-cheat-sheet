GCP Cloud Architect

Quick Reference & Certification Cheat Sheet

Security & Governance

DLP API Sensitive Data: Redact/Mask PII (Names, CCs) in text or images.
IAP Identity Aware Proxy: Access VMs/Web Apps via SSH/HTTPS without public IPs.
IAM Least Privilege: Use Predefined Roles by default. Use Custom Roles only if predefined roles are too broad.
IAM Hierarchy: Organization > Folders > Projects > Resources. (Permissions inherit downward).
VPC SC VPC Service Controls: Creates a perimeter to prevent data exfiltration (protects BQ, GCS).
Cloud Armor WAF/DDoS: IP Blacklisting, SQLi/XSS protection for Global Load Balancers.
Binary Auth GKE Security: Ensures only signed/trusted images are deployed to K8s.
Encryption Keys: CMEK (Customer Managed), CSEK (Customer Supplied), GMEK (Google Managed).
Workload Identity Federation: The secure way for AWS/Azure apps or GKE Pods to access GCP. Keyless authentication.

Data Pipelines & Messaging

Datastream Serverless CDC: Reads DB logs (Oracle Redo, MySQL Binlogs) with zero performance impact to stream changes to BQ/GCS. Google's cloud-native alternative to Oracle GoldenGate. Handles schema drift automatically.
Dataflow The Transformer: Apache Beam managed service. Unmatched power for heavy-duty batch/stream transformations. Do not use for simple CDC (requires custom query polling).
The Integration Trio:
1. Datastream (The Mover) mirrors operational DBs securely.
2. Dataflow (The Transformer) picks up the mirrored data to clean/format it.
3. Federated Queries (The Lens) lets BQ query external DBs on-the-fly without moving data.

Networking & Connectivity

VPC Global/Regional: VPCs are Global, Subnets are Regional.
Shared VPC Centralization: One host project manages a single network for multiple service projects, preventing IP conflicts.
VPC Peering Low Latency: Connects two distinct VPCs directly. Strictly impossible with overlapping CIDR ranges.
PSC Private Service Connect: Connects services via a single private IP endpoint without network interconnection (bypasses IP conflict issues).
Direct VPC Egress Serverless Native: Routes Cloud Run/Functions traffic directly to VPC without connector VMs.
Interconnect Hybrid: Dedicated (Physical line), Partner (via ISP), Cloud VPN (IPsec).
NCC Connectivity Center: Centralized hub-and-spoke transit network manager.
Cloud NAT Static IP Egress: Pair with Direct VPC Egress to give serverless a predictable static outbound IP.

Load Balancing

External App LB L7 (Global/Regional): Proxy-based. HTTP/S. Integrates with Cloud Armor, Cloud CDN, and IAP.
Internal App LB L7 (Regional): Envoy proxy-based. Internal private routing, traffic splitting (canary).
External Net LB L4 (Regional): Passthrough by default (terminates on VMs, preserves Client IP). Raw TCP/UDP.
Internal Net LB L4 (Regional): Passthrough. Exam feature: Supports Global Access (client VMs from *any* GCP region can access the internal LB).

Compute & Modernization

App Engine Standard PaaS: Scales to zero instantly. Restricted to specific language runtimes.
App Engine Flex Containers: Slower to scale, does not scale to zero. Custom Dockerfiles.
Cloud Run Serverless CaaS: Run stateless HTTP containers. Scales to zero. Supports concurrent requests.
Compute Engine (GCE) IaaS: Legacy apps, custom kernels, BYOL, or GPUs/TPUs.
Eventarc Event-Driven: Trigger Cloud Run/Functions from 90+ GCP sources (GCS, PubSub).
GKE Autopilot Serverless K8s: Google manages nodes; you pay for Pods used.
DaemonSet K8s Nodes: Ensures a specific Pod runs on all (or matching) Nodes (e.g., logging agents).
Migrate to Compute Engine Lift & Shift: Migrates VMs directly to GCE via data streaming with minimal downtime.
Exam Trap: Unpredictable traffic + cost optimization = App Engine Standard or Cloud Run (they scale to 0).

Hybrid & Multi-Cloud

Anthos / GKE Ent. Unified Platform: Manage K8s clusters across GCP, AWS, Azure, and on-prem.
Cross-Cloud Interconnect Networking: Dedicated physical connection between GCP and AWS, Azure, OCI.
ACM Anthos Config Mgmt: GitOps approach. Sync security, RBAC, and configs across fleets automatically.
GDC Distributed Cloud: Extends GCP hardware/services to edge or completely air-gapped environments.

Storage & Data

Object & Caching
GCS Classes Cost/Access: Standard (Hot), Nearline (< 1/mo), Coldline (< 1/qtr), Archive (< 1/yr).
Memorystore Caching: Managed Redis, Memcached, and Valkey. Sub-millisecond latency.
Operational & NoSQL
Cloud SQL HA Reliability: Synchronous replication across 2 zones. Auto-failover (Zero data loss).
Spanner Global Relational: Horizontally scalable, strongly consistent. Handles global HA automatically.
Bigtable NoSQL Key-Value: Petabyte-scale, high throughput (IoT, AdTech). Flat schema.
Firestore NoSQL Doc: Serverless, transactional, real-time sync. Ideal for Mobile/Web apps.
Governance & Migration
Knowledge Catalog Governance: (Dataplex) AI context graph to discover, secure, and monitor data lineage.
Storage Transfer Svc Cloud-to-Cloud: Automate transfers from AWS S3, Azure Blob, or HTTP to GCS.
Transfer Appliance Offline: Hardware server for high-volume physical data transfer (>10TB).

BigQuery & Analytics

Architecture & Optimization
Optimization Cost/Speed: Partitioning (Divide by Date/Int - scans less), Clustering (Sort within partitions - faster filters).
Security Views: Use Authorized Views to share results without granting raw table access.
Data Lakehouse & Ecosystem
Federated Queries The Lens: Query operational DBs (Cloud SQL, Spanner) in real time without moving data via `EXTERNAL_QUERY()`. Excellent for ad-hoc joins, but performance is bottlenecked by the source DB.
Cross-Cloud Lakehouse Omni: Execute distributed queries directly on AWS S3 or Azure Blob storage.
Lakehouse for Iceberg BigLake: Decouples compute and storage using open formats (Iceberg/Parquet) on GCS.
BigQuery Sharing Analytics Hub: Securely share datasets (Publisher/Subscriber model). Zero storage copy costs.
Advanced Analytics
Spanner Columnar Engine Performance: Accelerates federated queries up to 200x on live operational Spanner data.
BQML ML in SQL: Build models directly in BigQuery using standard SQL.

Deployments & Rollouts

Rolling Update Gradual: Slowly replaces old instances. Zero downtime, slower rollback.
Canary Risk Mitigation: Route a small percentage (e.g., 5%) to new version to test stability.
Blue/Green Instant Switch: Deploy new alongside old. Switch 100% via Load Balancer. Instant rollback.
A/B Testing Business Value: Route based on headers/cookies to measure conversion.

Disaster Recovery (DR)

RTO: Time to recover. RPO: Acceptable data loss.
Cold DR: High RTO. Restore from backups. Cheapest.
Warm DR: (Pilot Light) DB is sync'd, App scaled to zero.
Hot DR: Active-Active multi-region. Zero RTO/RPO. Expensive.

Architect Strategy & Traps

Decision Matrix:
- Need SQL + Global Scaling? ⮕ Spanner
- Need SQL + Vertical Scaling? ⮕ Cloud SQL
- Need Web Scale NoSQL? ⮕ Firestore
- Need Data Warehouse/Analytics? ⮕ BigQuery
- Need Hybrid Hub-and-Spoke Routing? ⮕ NCC
Ops Suite Monitoring: Cloud Logging (errors), Cloud Monitoring (metrics), Cloud Trace (latency).
Exam Traps:
- Datastream vs Dataflow: Don't build custom DB polling scripts in Dataflow. Use Datastream for zero-impact, log-based CDC replication.
- Transfer Appliance: Physical offline data (>10TB).
- Storage Transfer Svc: Cloud-to-Cloud (S3->GCS).
- Org Policy: Limits what resources can do. IAM limits who.
