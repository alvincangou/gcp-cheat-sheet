GCP Cloud Architect

Quick Reference & Certification Cheat Sheet

Security & Governance

DLP API Sensitive Data: Redact/Mask PII (Names, CCs) in text or images.
IAP Identity Aware Proxy: Access VMs/Web Apps via SSH/HTTPS without public IPs.
IAM Least Privilege: Use Predefined Roles by default. Use Custom Roles only if predefined roles are too broad (avoids admin overhead).
IAM Hierarchy: Organization > Folders > Projects > Resources. (Permissions inherit downward).
VPC SC VPC Service Controls: Creates a perimeter to prevent data exfiltration (protects BQ, GCS).
Cloud Armor WAF/DDoS: IP Blacklisting, SQLi/XSS protection for Global Load Balancers.
Binary Auth GKE Security: Ensures only signed/trusted images are deployed to K8s.
Encryption Keys: CMEK (Customer Managed), CSEK (Customer Supplied), GMEK (Google Managed - Default).
Workload Identity Federation: The secure way for AWS/Azure apps or GKE Pods to access GCP services. Keyless authentication (avoids downloading/rotating long-lived JSON service account keys).

Networking & Connectivity

VPC Global/Regional: VPCs are Global, Subnets are Regional.
Shared VPC Centralization: One host project manages a single network for multiple service projects, preventing IP conflicts at the source through centralized administration.
VPC Peering Low Latency: Connects two distinct VPCs directly (non-transitive). Strictly impossible if there are overlapping (conflicting) CIDR ranges.
PSC (Private Service Connect) Network Abstraction: Connects services (Google APIs, 3rd-party SaaS, or internal apps) via a single private IP endpoint without network interconnection, completely bypassing IP conflict issues.
Direct VPC Egress Serverless Native: Routes Cloud Run/Functions traffic directly to VPC without connector VMs (lower latency, higher throughput, scales instantly).
VPC Connector Serverless Legacy: Uses managed underlying VMs to bridge serverless to VPC (has throughput caps, minimum cost, slower scaling).
Interconnect Hybrid: Dedicated (Physical line), Partner (via ISP), Cloud VPN (IPsec over public web).
Cloud NAT Static IP Egress: Pair with Direct VPC Egress to give serverless workloads a predictable, static outbound public IP.
Cloud DNS Resolution: Use 169.254.169.254 for metadata/internal resolution.

Load Balancing

External App LB L7 (Global/Regional): Proxy-based. Handles HTTP/S. Integrates with Cloud Armor, Cloud CDN, and IAP. Features URL-based routing and SSL termination.
Internal App LB L7 (Regional): Envoy proxy-based. Internal private routing, traffic splitting (canary deploys), and HTTP header manipulation for private microservices.
External Net LB L4 (Regional): Passthrough by default (terminates on VMs, preserves Client IP). Used for raw TCP/UDP/ESP. Can also be configured as a Proxy (TCP/SSL Proxy).
Internal Net LB L4 (Regional): Passthrough, ultra-low latency internal routing. Crucial exam feature: Supports Global Access (allows client VMs from *any* GCP region to access the internal LB).

Compute & Modernization

App Engine Standard PaaS: Scales to zero instantly, bills by milliseconds. Restricted to specific language runtimes. Ideal for sudden traffic spikes.
App Engine Flex Containers: Uses underlying Compute Engine VMs. Slower to scale, does not scale to zero. Allows custom Dockerfiles and background threads.
Cloud Run Serverless CaaS: Run any stateless HTTP container. Scales to zero. Supports concurrent requests per instance (cheaper than Cloud Functions for high traffic).
Compute Engine (GCE) IaaS: Use for legacy apps, custom kernels, strict licensing (BYOL), or specialized hardware (GPUs/TPUs).
Istio Service Mesh: Traffic management, security, and observability for microservices.
Eventarc Event-Driven: Trigger Cloud Run/Functions from 90+ GCP sources (GCS, PubSub).
GKE Autopilot Serverless K8s: Google manages nodes; you pay for Pods used.Best for teams that want K8s without infrastructure overhead.
HPA / CA Scaling: HPA (more pods), CA (more nodes), VPA (bigger pods).
K8s Namespace Isolation: Logical environments; use ResourceQuotas to limit CPU/RAM per team.
DaemonSet K8s Nodes: Ensures a specific Pod runs on all (or matching) Nodes. Useful for logging or monitoring agents.
ReplicaSet K8s Pods: Maintains a stable set of replica Pods running at any given time (usually managed by Deployments).
Migrate to Compute Engine Lift & Shift: Migrates physical/virtual VMs (AWS, Azure, VMware) directly to GCE via data streaming with minimal downtime.
Migrate to Containers Modernization: Automatically extracts application workloads from legacy VMs and converts them into container images for GKE or Cloud Run.
Exam Trap: If the prompt mentions "unpredictable traffic" + "cost optimization", look for App Engine Standard or Cloud Run (because they scale to 0).

Hybrid & Multi-Cloud

Anthos / GKE Ent. Unified Platform: Manage K8s clusters across GCP, AWS, Azure, and on-prem (Bare Metal/VMware) from a single pane of glass.
BigQuery Omni Multi-Cloud Analytics: Query data residing in AWS S3 or Azure Blob storage using standard SQL without moving or copying the data to GCP.
Cross-Cloud Interconnect Networking: A direct, high-bandwidth, dedicated physical connection between GCP and other cloud providers (AWS, Azure, OCI, Alibaba).
ACM Anthos Config Mgmt: GitOps approach. Sync security policies, RBAC, and cluster configs across your entire multi-cloud fleet automatically.
Connect Gateway Secure Access: Securely interact with registered external clusters via Google Cloud ID, bypassing the need for inbound firewall rules or public IPs.
Cloud Service Mesh Traffic/Security: Managed Istio. Provides mTLS encryption, traffic splitting, and unified telemetry across clusters spanning different clouds.
GDC (Distributed Cloud) Edge & Sovereign: Extends GCP hardware and services to on-prem, edge, or completely air-gapped (disconnected) environments to meet strict data sovereignty or local processing needs.

Deployments & Rollouts

Rolling Update Gradual: Slowly replaces old instances with new ones. Default K8s strategy. Zero downtime, but slower rollback.
Canary Risk Mitigation: Route a small percentage of live traffic (e.g., 5%) to the new version to test stability before full rollout.
Blue/Green Instant Switch: Deploy the new version (Green) alongside the old (Blue). Switch traffic 100% via Load Balancer. Instant rollback.
A/B Testing Business Value: Route specific users to a new version based on HTTP headers/cookies to measure conversion or user behavior.
Shadow Safe Testing: Mirror live production traffic to the new version without affecting end users to test scale and errors.

Storage & Data

GCSFUSE File Mount: Mount GCS buckets as local file systems on Linux.
GCS Classes Cost/Access:
  • Standard: Hot data.
  • Nearline: Accessed < 1/month (30 day min).
  • Coldline: Accessed < 1/quarter (90 day min).
  • Archive: Accessed < 1/year (365 day min).
Bigtable NoSQL: Petabyte-scale, high throughput (IoT, AdTech). Flat schema. Supports native Vector Search for real-time GenAI grounding.
Firestore NoSQL Doc: Serverless, transactional, real-time sync. Ideal for Mobile/Web apps. Supports native Vector Search (embeddings/RAG workloads).
Vs Bigtable: Use Firestore for hierarchical docs/apps (ACID, real-time sync, mobile, user profile vector store). Use Bigtable for massive ingest/flat time-series (IoT/Analytics, high-volume operational vectors).
Memorystore Caching: Managed Redis, Memcached, and Valkey (open-source alternative). Sub-millisecond latency for key-value caching.
Knowledge Catalog Governance: (Formerly Dataplex Universal Catalog). Gemini-powered active AI context graph to discover, secure, and monitor data across GCS/BigQuery. Serves as an enterprise truth engine for AI agents.
Vs VPC-SC: Knowledge Catalog / Dataplex governs metadata discovery, data quality, and lineage (L7). VPC Service Controls creates a strict cryptographic network perimeter to prevent data exfiltration (L4).
DMS DB Migration: Serverless migration for MySQL, PostgreSQL, SQL Server (Cloud SQL/AlloyDB).
Storage Transfer Svc Cloud-to-Cloud: Automate transfers from AWS S3, Azure Blob, file systems, or HTTP to GCS. Scalable online migration.
Transfer Appliance Offline: Hardware server for high-volume physical data transfer (>10TB to hundreds of TBs).
Cloud SQL HA Reliability: Synchronous replication across two zones in the same region. Standby instance for auto-failover. Enterprise Plus tier adds sub-second maintenance downtime and read data caching.
Cloud SQL Replicas Performance: Asynchronous replication. Used to offload heavy read traffic or for cross-region Disaster Recovery (requires manual promotion).
Spanner Global Relational: Horizontally scalable, strongly consistent. Now supports Spanner Graph (SQL + GQL query integration). Fine-grained sizing via Processing Units (PUs) removes strict 1-node minimum constraints for smaller apps.

BigQuery & Analytics

Serverless DW Architecture: Decoupled storage (Colossus) & compute (Dremel). SQL Interface.
Optimization Cost/Speed:
  • Partitioning: Divide table by Date/Time/Int (Scans less data).
  • Clustering: Sort data within partitions (Faster filters).
Security Views: Use Authorized Views to share query results without granting access to underlying raw tables.
BQML ML in SQL: Build models (Linear Reg, K-Means) directly in BigQuery using standard SQL.
Federated External: Query data residing in GCS, Bigtable, or SQL (Cloud SQL) without moving it.

Disaster Recovery (DR)

RTO: Time to recover. RPO: Acceptable data loss.
Cold DR: High RTO. Restore from backups. Cheapest.
Warm DR: (Pilot Light) Database is sync'd, App is scaled to zero.
Hot DR: Active-Active multi-region. Zero RTO/RPO. Expensive.

Architect Strategy & Traps

Decision Matrix:
- Need SQL + Global Scaling? ⮕ Spanner
- Need SQL + Vertical Scaling? ⮕ Cloud SQL
- Need Web Scale NoSQL? ⮕ Firestore
- Need Hadoop/Spark? ⮕ Dataproc
- Need Data Warehouse/Analytics? ⮕ BigQuery
Ops Suite Monitoring: Cloud Logging (errors), Cloud Monitoring (metrics), Cloud Trace (latency), Cloud Debugger.
Exam Traps:
- Transfer Appliance: Use for physical offline data (>10TB).
- Storage Transfer Svc: Use for Cloud-to-Cloud (S3->GCS).
- Org Policy: Limits what resources can do. IAM limits who.
