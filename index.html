GCP Cloud Architect

Quick Reference & Certification Cheat Sheet

Security & Governance

DLP API Sensitive Data: Redact/Mask PII (Names, CCs) in text or images.
IAP Identity Aware Proxy: Access VMs/Web Apps via SSH/HTTPS without public IPs.
IAM Least Privilege: Use Predefined Roles by default. Use Custom Roles only if predefined roles are too broad (avoids admin overhead).
IAM Hierarchy: Organization > Folders > Projects > Resources. (Permissions inherit downward).
VPC SC VPC Service Controls: Creates a perimeter to prevent data exfiltration (protects BQ, GCS).
Cloud Armor WAF/DDoS: IP Blacklisting, SQLi/XSS protection for Global Load Balancers.
Binary Auth GKE Security: Ensures only signed/trusted images are deployed to K8s.(th trust signal is given by Artifact Analysis)
Encryption Keys: CMEK (Customer Managed), CSEK (Customer Supplied), GMEK (Google Managed - Default).
Workload Identity Federation: The secure way for AWS/Azure apps or GKE Pods to access GCP services. Keyless authentication (avoids downloading/rotating long-lived JSON service account keys).

Datastream Serverless CDC: Reads DB logs (Oracle Redo, MySQL Binlogs) with zero performance impact to stream changes to BQ/GCS. Google's cloud-native alternative to Oracle GoldenGate. Handles schema drift automatically.
Dataflow The Transformer: Apache Beam managed service. Unmatched power for heavy-duty batch/stream transformations. Do not use for simple CDC (requires custom query polling).
Decision Tip: "Group data by time window" or "Unified Batch/Stream" = Dataflow. "Buffer messages" = Pub/Sub. The Integration Trio:
1. Datastream (The Mover) mirrors operational DBs securely.
2. Dataflow (The Transformer) picks up the mirrored data to clean/format it.
3. Federated Queries (The Lens) lets BQ query external DBs on-the-fly without moving data.

Networking & Connectivity

VPC Global/Regional: VPCs are Global, Subnets are Regional.
Shared VPC Centralization: One host project manages a single network for multiple service projects, preventing IP conflicts at the source through centralized administration.
VPC Peering Low Latency: Connects two distinct VPCs directly (non-transitive). Strictly impossible if there are overlapping (conflicting) CIDR ranges.
PSC (Private Service Connect) Network Abstraction: Connects services (Google APIs, 3rd-party SaaS, or internal apps) via a single private IP endpoint without network interconnection, completely bypassing IP conflict issues.
Direct VPC Egress Serverless Native: Routes Cloud Run/Functions traffic directly to VPC without connector VMs (lower latency, higher throughput, scales instantly).
VPC Connector Serverless Legacy: Uses managed underlying VMs to bridge serverless to VPC (has throughput caps, minimum cost, slower scaling).
Interconnect Hybrid: Dedicated (Physical line), Partner (via ISP), Cloud VPN (IPsec over public web).
NCC Connectivity Center: Centralized hub-and-spoke transit network manager. Seamlessly connects VPCs, Cloud VPN, Interconnect, and 3rd-party SD-WAN/Router Appliances.
Cloud NAT Static IP Egress: Pair with Direct VPC Egress to give serverless workloads a predictable, static outbound public IP.
Cloud DNS Resolution: Use 169.254.169.254 for metadata/internal resolution.

Load Balancing

External App LB L7 (Global/Regional): Proxy-based. Handles HTTP/S. Integrates with Cloud Armor, Cloud CDN, and IAP. Features URL-based routing and SSL termination.
Internal App LB L7 (Regional): Envoy proxy-based. Internal private routing, traffic splitting (canary deploys), and HTTP header manipulation for private microservices.
External Net LB L4 (Regional): Passthrough by default (terminates on VMs, preserves Client IP). Used for raw TCP/UDP/ESP. Can also be configured as a Proxy (TCP/SSL Proxy).
Internal Net LB L4 (Regional): Passthrough, ultra-low latency internal routing. Crucial exam feature: Supports Global Access (allows client VMs from *any* GCP region to access the internal LB).

Compute & Modernization

App Engine Standard PaaS: Scales to zero instantly, bills by milliseconds. Restricted to specific language runtimes. Ideal for sudden traffic spikes.
App Engine Flex Containers: Uses underlying Compute Engine VMs. Slower to scale, does not scale to zero. Allows custom Dockerfiles and background threads.
Cloud Run Serverless CaaS: Run any stateless HTTP container. Scales to zero. Supports concurrent requests per instance (cheaper than Cloud Functions for high traffic).
Compute Engine (GCE) IaaS: Use for legacy apps, custom kernels, strict licensing (BYOL), or specialized hardware (GPUs/TPUs).
Spot VMs Termination Notice: Provides a 30-second signal (via ACPI/Metadata) before reclaiming the instance. Use a Shutdown Script to safely save state, flush logs, or drain connections.
GCE Disks Local SSD vs PD: Local SSDs are ephemeral (physically attached, highest IOPS, data is lost if VM stops). Persistent Disks (PD) are durable network storage (survive VM stops, lower IOPS).
Hyperdisk Next-Gen Block: Decouples performance (IOPS/throughput) from disk capacity. Extreme (Max IOPS for databases), Throughput (Scale-out scale), Balanced (Modern boot/application default).
Istio Service Mesh: Traffic management, security, and observability for microservices.
Eventarc Event-Driven: Trigger Cloud Run/Functions from 90+ GCP sources (GCS, PubSub).
GKE Autopilot Serverless K8s: Google manages nodes; you pay for Pods used.Best for teams that want K8s without infrastructure overhead.
HPA / CA Scaling: HPA (more pods), CA (more nodes), VPA (bigger pods).
K8s Namespace Isolation: Logical environments; use ResourceQuotas to limit CPU/RAM per team.
DaemonSet K8s Nodes: Ensures a specific Pod runs on all (or matching) Nodes. Useful for logging or monitoring agents.
ReplicaSet K8s Pods: Maintains a stable set of replica Pods running at any given time (usually managed by Deployments).
Migrate to Compute Engine Lift & Shift: Migrates physical/virtual VMs (AWS, Azure, VMware) directly to GCE via data streaming with minimal downtime.
Migrate to Containers Modernization: Automatically extracts application workloads from legacy VMs and converts them into container images for GKE or Cloud Run.
Exam Trap: If the prompt mentions "unpredictable traffic" + "cost optimization", look for App Engine Standard or Cloud Run (because they scale to 0).

Hybrid & Multi-Cloud

Anthos / GKE Ent. Unified Platform: Manage K8s clusters across GCP, AWS, Azure, and on-prem (Bare Metal/VMware) from a single pane of glass.
Cross-Cloud Interconnect Networking: A direct, high-bandwidth, dedicated physical connection between GCP and other cloud providers (AWS, Azure, OCI, Alibaba).
ACM Anthos Config Mgmt: GitOps approach. Sync security policies, RBAC, and cluster configs across your entire multi-cloud fleet automatically.
Connect Gateway Secure Access: Securely interact with registered external clusters via Google Cloud ID, bypassing the need for inbound firewall rules or public IPs.
Cloud Service Mesh Traffic/Security: Managed Istio. Provides mTLS encryption, traffic splitting, and unified telemetry across clusters spanning different clouds.
GDC (Distributed Cloud) Edge & Sovereign: Extends GCP hardware and services to on-prem, edge, or completely air-gapped (disconnected) environments to meet strict data sovereignty or local processing needs.

Storage & Data

Object, File Storage & Caching
GCSFUSE File Mount: Mount GCS buckets as local file systems on Linux.
Lustre / Parallelstore HPC File System: Ultra-high performance parallel file system for scratch space, AI/ML (GPUs), and High-Performance Computing. Delivers sub-millisecond latency and massive IOPS.
GCS Classes Cost/Access:
  • Standard: Hot data.
  • Nearline: Accessed < 1/month (30 day min).
  • Coldline: Accessed < 1/quarter (90 day min).
  • Archive: Accessed < 1/year (365 day min).
Memorystore Caching: Managed Redis, Memcached, and Valkey (open-source alternative). Sub-millisecond latency for key-value caching.
Operational & NoSQL Databases
Cloud SQL HA Reliability: Synchronous replication across two zones in the same region. Standby instance for auto-failover (Zero data loss). Enterprise Plus tier adds sub-second maintenance downtime and read data caching.
Cloud SQL Replicas Performance: Asynchronous replication. Used to offload heavy read traffic or for cross-region Disaster Recovery (requires manual promotion).
Spanner Global Relational: Horizontally scalable, strongly consistent relational database. Handles global HA automatically. Supports Spanner Graph (SQL + GQL query integration). Fine-grained sizing via Processing Units (PUs) removes strict 1-node minimum constraints.
Bigtable NoSQL Key-Value: Petabyte-scale, high throughput (IoT, AdTech). Flat schema. Supports native Vector Search for real-time GenAI grounding.
Firestore NoSQL Doc: Serverless, transactional, real-time sync. Ideal for Mobile/Web apps. Supports native Vector Search (embeddings/RAG workloads).
Vs Bigtable: Use Firestore for hierarchical docs/apps (ACID, real-time sync, mobile, user profile vector store). Use Bigtable for massive ingest/flat time-series (IoT/Analytics, high-volume operational vectors).
Governance, Security & Migration
Knowledge Catalog Governance: (Formerly Dataplex Universal Catalog). Gemini-powered active AI context graph to discover, secure, and monitor data quality/lineage across GCS and BigQuery. Serves as an enterprise truth engine for AI agents.
Vs VPC-SC: Knowledge Catalog / Dataplex governs metadata discovery, data quality, and lineage (L7). VPC Service Controls creates a strict cryptographic network perimeter to prevent data exfiltration (L4).
DMS DB Migration: Serverless migration for MySQL, PostgreSQL, SQL Server (Cloud SQL/AlloyDB).
Storage Transfer Svc Cloud-to-Cloud: Automate transfers from AWS S3, Azure Blob, file systems, or HTTP to GCS. Scalable online migration.
Storage Transfer Svc On-Prem Agent: Uses a local software agent to securely migrate massive file archives (billions of files) from on-prem data centers to GCS over an Interconnect or VPN.
Transfer Appliance Offline: Hardware server for high-volume physical data transfer (>10TB to hundreds of TBs).

BigQuery & Analytics

Architecture & Optimization
Serverless DW Architecture: Decoupled storage (Colossus) & compute (Dremel). SQL Interface.
Optimization Cost/Speed:
  • Partitioning: Divide table by Date/Time/Int (Scans less data).(always priorize partioning over clustering)
  • Clustering: Sort data within partitions (Faster filters).
Security Views: Use Authorized Views to share query results without granting access to underlying raw tables.
Data Lakehouse & Ecosystem
The BigQuery Quadrant (Compute vs. Storage Architecture):
1. Federated: Query operational DBs (Cloud SQL) & external files in real time.
2. Cross-Cloud: (Formerly BQ Omni) Query AWS/Azure storage without moving data.
3. Lakehouse for Iceberg: (Formerly BigLake) Query open-format GCS files like native tables.
4. Sharing: (Formerly Analytics Hub) Share internal BQ assets securely with zero data copying.
Federated Queries Real-time Hybrid & External: Query operational databases (Cloud SQL, Spanner, AlloyDB) or data residing directly in GCS and Bigtable in real time without moving or copying data. Uses EXTERNAL_QUERY() or External Datasets mapping.
• Best For: Low-volume, real-time ad-hoc operational reporting where data changes too fast for ETL.
• Caveat: Puts analytical query load directly on your production transactional database.
Cross-Cloud Lakehouse Multi-Cloud Analytics: (Formerly BigQuery Omni). Execute distributed BigQuery queries directly on AWS S3 or Azure Blob storage using standard SQL without moving or copying the data to GCP.
• Best For: Overcoming "data gravity" when moving multi-terabyte datasets across clouds is too slow or introduces massive egress costs.
Lakehouse for Apache Iceberg Open Storage Engine: (Formerly BigLake). Decouples BigQuery's compute engine from storage using open metadata and formats (Iceberg/Parquet/Avro) on GCS with fine-grained access control.
• Architecture Note: Not a BigQuery alternative, but a storage capability of BigQuery. BQ acts as the query engine executing over open-source table formats.
• Best For: Multi-engine environments (e.g., sharing the same files seamlessly with Spark or Trino) and strictly avoiding vendor lock-in. *Use Native BQ Storage if maximum query speed is your single priority.*
BigQuery Sharing Data Exchange: (Formerly Analytics Hub). Secure platform to share BigQuery datasets, ML models, and AI assets inside or outside the organization without data duplication.
• Best For: Data monetization, B2B data sharing, or cross-department analytics without managing complex ETL pipelines or access tokens.
Publisher-Subscriber Model: Publishers create Exchanges and Listings. Subscribers access them via read-only Linked Datasets. Subscriber pays only for query compute; zero storage copy costs.
Advanced Analytics & Performance
Spanner Columnar Engine Performance: Automatically accelerates federated queries and analytical scans up to 200x on live operational Spanner data using vectorized execution.
BQML ML in SQL: Build models (Linear Reg, K-Means) directly in BigQuery using standard SQL.

Deployments & Rollouts

Rolling Update Gradual: Slowly replaces old instances with new ones. Default K8s strategy. Zero downtime, but slower rollback.
Canary Risk Mitigation: Route a small percentage of live traffic (e.g., 5%) to the new version to test stability before full rollout.
Blue/Green Instant Switch: Deploy the new version (Green) alongside the old (Blue). Switch traffic 100% via Load Balancer. Instant rollback.
A/B Testing Business Value: Route specific users to a new version based on HTTP headers/cookies to measure conversion or user behavior.
Shadow Safe Testing: Mirror live production traffic to the new version without affecting end users to test scale and errors.

Disaster Recovery (DR)

RTO: Time to recover. RPO: Acceptable data loss.
Cold DR: High RTO. Restore from backups. Cheapest.
Warm DR: (Pilot Light) Database is sync'd, App is scaled to zero.
Hot DR: Active-Active multi-region. Zero RTO/RPO. Expensive.

Architect Strategy & Traps

Decision Matrix:
- Need SQL + Global Scaling? ⮕ Spanner
- Need SQL + Vertical Scaling? ⮕ Cloud SQL
- Need Web Scale NoSQL? ⮕ Firestore
- Need Hadoop/Spark? ⮕ Dataproc
- Need Data Warehouse/Analytics? ⮕ BigQuery
- Need Hybrid Hub-and-Spoke / Transit Routing? ⮕ NCC
Ops Suite Observability: Cloud Logging (logs), Cloud Monitoring (metrics/dashboards), Cloud Trace (request latency), Error Reporting (aggregates app crashes), and Cloud Profiler (analyzes CPU/RAM performance in production).
Exam Traps:
- Datastream vs Dataflow: Don't build custom DB polling scripts in Dataflow. Use Datastream for zero-impact, log-based CDC replication.
- Transfer Appliance: Use for physical offline data (>10TB).
- Storage Transfer Svc: Use for Cloud-to-Cloud (S3->GCS).
- Org Policy: Limits what resources can do. IAM limits who.

1. Altostrat Media (Media & Generative AI)

A major media company modernizing content management and user engagement using Generative AI while managing growing storage costs and integrating legacy on-premises ingestion pipelines.

Key Requirements

  • Business: Accelerate workflows across hybrid pipelines; 24/7 conversational commerce; optimize long-term storage costs for archival media assets; extract deep analytical trends/metadata from audio/video files.
  • Technical: Unified containerized CI/CD framework across hybrid footprints; automated GenAI content workflows (tagging, text summaries); secure high-throughput path for massive on-prem ingestion.

Proposed Architectural Solution

GKE Enterprise (Anthos) Consistent on-prem/GCP container environments and unified GitOps configuration sync via code repos.
Vertex AI Gemini Models Standardizes multimodal understanding to auto-summarize audio/video and extract rich semantic metadata fields.
Dialogflow CX + RAG Powers 24/7 client conversational customer assistance via private internal vector groundings.
GCS Lifecycle Policies Configures automated bucket tracking rules to offload raw clips from Standard ⮕ Nearline ⮕ Coldline/Archive.
Cloud Run Functions Serves as low-overhead event-driven serverless executor triggered instantly upon new video storage landing events.
Security Core CMEK via Cloud KMS for revocation control; VPC Service Controls (VPC-SC) to prevent asset data exfiltration.
Exam Traps & Pitfalls:
- Storage Pitfall: Moving active media directly to Coldline/Archive. Trap fix: Active media must remain in Standard/Nearline to protect from high egress access fees.
- Compute Pitfall: Provisioning manual GCE VMs for transcoding pipelines. Trap fix: Introduces idle resource overhead; stick to millisecond-billed serverless Cloud Run Functions instead.
- Governance Pitfall: Default Google-Managed Keys (GMEK) for highly confidential assets. Trap fix: Fails independent external audit rules; client-controlled CMEK is required.
2. Cymbal Retail (E-Commerce & DB Modernization)

An expanding retail brand automating catalog enrichment, personalized customer search, and conversational commerce while modernizing a hybrid multi-database stack.

Key Requirements

  • Business: Automate product data enrichment; elevate storefront search conversions; eliminate manual entry overhead; enforce supply chain code scanning before container deployments.
  • Technical: Securely expose internal APIs to external supply chain networks; mask column-level customer PII from unauthorized analysts; low-latency ephemeral scratch space for custom AI model adjustments.

Proposed Architectural Solution

Local SSDs (NVMe) Physically attached high-IOPS storage slots to support fast, ephemeral scratch space for custom ML container runs.
Apigee + PSC Publishes private endpoints securely to partners without setting up full-scale network route integrations or exposing public IPs.
Retail Search (Discovery AI) Delivers native semantic retail intent engines for personalized customer discovery workflows.
Firestore Serverless, horizontally scalable NoSQL document architecture that effortlessly replaces on-prem MongoDB instances.
Cloud SQL / AlloyDB Fully managed drop-in replacements for transactional legacy relational instances (MySQL, MS SQL Server).
Supply Chain Sec Cloud Build + Artifact Registry + Binary Authorization to cryptographically guarantee image security before production runs.
Exam Traps & Pitfalls:
- Storage Pitfall: Writing raw active ML scratch data directly to basic GCS network buckets. Trap fix: Introduces severe pipeline latency loops; physically attached Local SSDs must be used.
- Networking Pitfall: Setting up standard VPC Peering connections for external partners. Trap fix: Exposes full networks and risks overlapping CIDR blocks; isolate interactions completely via Apigee and Private Service Connect (PSC).
3. EHR Healthcare (SaaS & High-Availability DR)

A critical medical software provider migrating legacy collocated environments to GCP while guaranteeing strict regulatory compliance, a 99.99% infrastructure availability SLA, and zero system downtime.

Key Requirements

  • Business: Achieve a strict 99.99% architecture SLA; maintain complete HIPAA, HITECH, and HITRUST compliance; ensure a Recovery Time Objective (RTO) of under 30 minutes for core databases.
  • Technical: Redundant network routing pipelines; eliminate local configuration drift globally via automated GitOps engines; mask patient clinical PHI out of downstream big-data targets.

Proposed Architectural Solution

Dedicated Interconnect Configured in an Active-Active Multi-Metro design structure to safely fulfill Google's 99.99% core connection SLA metrics.
GKE Ent. + Config Sync Establishes continuous declarative GitOps configurations across clusters to completely bypass localized network configuration changes.
Cloud SQL for SQL Server Deployed with cross-region asynchronous replicas ready for manual promotion scripts to achieve an RTO under 30 minutes.
Sensitive Data Protection Automatically scans, identifies, and obscures Patient Health Information (PHI) logs via Cloud DLP rules before storage.
Vertex AI Private Tuning Utilizes foundation parameters backed by VPC Service Controls perimeter definitions to protect patient info from public lookups.
Exam Traps & Pitfalls:
- Network Pitfall: Relying on standard High-Availability Cloud VPN tunnels for critical links. Trap fix: Cloud VPN operates over public transit nodes and maxes out at a 99.9% SLA, breaking the business requirement.
- DR Pitfall: Shipping nightly flat database backups to standard GCS buckets. Trap fix: Restoring massive operational environments from scratch takes hours, failing the 30-minute RTO requirement; use active cross-region replicas. - Encryption Pitfall: Assuming CMEK is contractually mandatory for baseline HIPAA compliance. Trap fix: GMEK is enough under GCP's standard Business Associate Agreement (BAA). Only suggest CMEK if independent, external key-revocation control is explicitly specified.
4. KnightMotives Automotive (IoT & Global Consistency)

A vehicle manufacturer upgrading telemetry systems to support live vehicle telemetry tracking, global inventory sync, and secure data sharing ecosystems.

Key Requirements

  • Business: Modernize unoptimized order flows with real-time inventory checks; securely share anonymous vehicle performance patterns with insurance partners; build a 100% cloud-based web ecosystem for dealers.
  • Technical: Ingest and process high-frequency time-series logging feeds from millions of global vehicles; maintain absolute transactional ACID consistency globally to block duplicate VIN bookings.

Proposed Architectural Solution

Cloud Bigtable The standard choice for time-series stream data; horizontally processes millions of low-latency tracking writes.
Pub/Sub + Dataflow Ingests vehicle messaging queues and applies structured streaming cleanup before cold warehouse data drops.
Cloud Spanner Powers order configurations, leveraging internal TrueTime clocks to preserve global multi-region ACID consistency.
BigQuery Analytics Hub Securely exposes clean data subsets directly to insurance companies via read-only Linked Datasets, bypassing data replication costs.
Cloud Run Hosts web-based portal configurations that scale to zero, minimizing dealer overhead with zero hardware (CAPEX) requirements.
Exam Traps & Pitfalls:
- Database Pitfall: Dropping high-speed live engine telemetry directly into Cloud SQL relational setups. Trap fix: Relational write locks break completely under massive IoT scales; telemetry always belongs in **Cloud Bigtable**.
- Consistency Pitfall: Deploying local regional databases with manual peering updates for inventory sync. Trap fix: Introduces replication lag that causes duplicate sales or double-bookings; **Cloud Spanner** is required for global ACID guarantees.
