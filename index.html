GCP Cloud Architect

Quick Reference & Certification Cheat Sheet

Security & Governance

DLP API Sensitive Data: Redact/Mask PII (Names, CCs) in text or images.
IAP Identity Aware Proxy: Access VMs/Web Apps via SSH/HTTPS without public IPs.
IAM Least Privilege: Use Custom Roles. Hierarchy: Org > Folders > Projects > Resources.
VPC SC VPC Service Controls: Creates a perimeter to prevent data exfiltration (protects BQ, GCS).
Cloud Armor WAF/DDoS: IP Blacklisting, SQLi/XSS protection for Global Load Balancers.
Binary Auth GKE Security: Ensures only signed/trusted images are deployed to K8s.
Encryption Keys: CMEK (Customer Managed), CSEK (Customer Supplied), GMEK (Google Managed).

Networking & Connectivity

VPC Global/Regional: VPCs are Global, Subnets are Regional.
Shared VPC Centralization: One host project manages network for multiple service projects.
VPC Peering Low Latency: Connect two VPCs directly (non-transitive).
PSC Private Service Connect: Access Google APIs or 3rd party services via private IPs.
Interconnect Hybrid: Dedicated (Physical line), Partner (via ISP), Cloud VPN (IPsec over public web).
Cloud DNS Resolution: Use 169.254.169.254 for metadata/internal resolution.

Load Balancing

Global HTTP(S) Layer 7: Web Apps, CDN, Cloud Armor, Multi-region failover.
Network LB Layer 4: Regional. TCP/UDP traffic (non-HTTP).
Internal LB Private: Use for internal microservices tiers.

Compute & Modernization

Istio Service Mesh: Traffic management, security, and observability for microservices.
Eventarc Event-Driven: Trigger Cloud Run/Functions from 90+ GCP sources (GCS, PubSub).
GKE Autopilot Serverless K8s: Google manages nodes; you pay for Pods used.
HPA / CA Scaling: HPA (more pods), CA (more nodes), VPA (bigger pods).
K8s Namespace Isolation: Logical environments; use ResourceQuotas to limit CPU/RAM per team.
Migrate for Compute VM Lift & Shift: Migrate VMs (VMware/AWS/Azure) to GCE with streaming replication & minimal downtime.
ACM Anthos Config Mgmt: GitOps for Hybrid/Multi-cloud. Sync policies & configs across fleets.
Connect Gateway Hybrid Access: Securely access registered Anthos clusters via Google Cloud ID without public IPs.

Storage & Data

GCSFUSE File Mount: Mount GCS buckets as local file systems on Linux.
GCS Classes Cost/Access:
  • Standard: Hot data.
  • Nearline: Accessed < 1/month (30 day min).
  • Coldline: Accessed < 1/quarter (90 day min).
  • Archive: Accessed < 1/year (365 day min).
Bigtable NoSQL: Petabyte-scale, high throughput (IoT, AdTech). Flat schema.
Firestore NoSQL Doc: Serverless, transactional, real-time sync. Ideal for Mobile/Web apps.
Vs Bigtable: Use Firestore for hierarchical documents/apps (ACID). Use Bigtable for massive ingest/flat data (IoT/Analytics).
Memorystore Caching: Managed Redis/Memcached. Sub-millisecond latency.
DMS DB Migration: Serverless migration for MySQL, PostgreSQL, SQL Server (Cloud SQL/AlloyDB).
Storage Transfer Svc Cloud-to-Cloud: Automate transfers from AWS S3, Azure Blob, or HTTP to GCS. Scalable online migration.
Transfer Appliance Offline: Hardware server for high-volume physical data transfer (>10TB).

BigQuery & Analytics

Serverless DW Architecture: Decoupled storage (Colossus) & compute (Dremel). SQL Interface.
Optimization Cost/Speed:
  • Partitioning: Divide table by Date/Time/Int (Scans less data).
  • Clustering: Sort data within partitions (Faster filters).
Security Views: Use Authorized Views to share query results without granting access to underlying raw tables.
BQML ML in SQL: Build models (Linear Reg, K-Means) directly in BigQuery using standard SQL.
Federated External: Query data residing in GCS, Bigtable, or SQL (Cloud SQL) without moving it.

Disaster Recovery (DR)

RTO: Time to recover. RPO: Acceptable data loss.
Cold DR: High RTO. Restore from backups. Cheapest.
Warm DR: (Pilot Light) Database is sync'd, App is scaled to zero.
Hot DR: Active-Active multi-region. Zero RTO/RPO. Expensive.

Architect Strategy & Traps

Decision Matrix:
- Need SQL + Global Scaling? ⮕ Spanner
- Need SQL + Vertical Scaling? ⮕ Cloud SQL
- Need Web Scale NoSQL? ⮕ Firestore
- Need Hadoop/Spark? ⮕ Dataproc
- Need Data Warehouse/Analytics? ⮕ BigQuery
Ops Suite Monitoring: Cloud Logging (errors), Cloud Monitoring (metrics), Cloud Trace (latency), Cloud Debugger.
Exam Traps:
- Transfer Appliance: Use for physical offline data (>10TB).
- Storage Transfer Svc: Use for Cloud-to-Cloud (S3->GCS).
- Org Policy: Limits what resources can do. IAM limits who.
